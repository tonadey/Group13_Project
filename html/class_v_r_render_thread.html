Detailed Description

Owns the VTK/OpenVR render loop on a worker thread.

The desktop Qt UI remains responsive while this thread manages the headset renderer, pending actor queue, scene clearing, and simple rotation commands.

Member Enumeration Documentation

◆ Command

enum VRRenderThread::Command

Send a command to the VR thread in a thread-safe way. The thread applies it on its next frame.

END_RENDER asks the thread to exit run() cleanly. ROTATE_X / Y / Z set a per-tick rotation in degrees applied to every actor in the scene every ~20ms (set value to 0 to stop). CLEAR_SCENE drops all actors currently in the VR renderer; used by the GUI's Sync button to push a fresh snapshot of the tree (e.g. after the user changed a filter slider).

bool VRRenderThread::isVRAvailable ( QString * reason = nullptr)

Checks whether OpenVR can start before the render thread is launched.

Pre-flight check before launching the VR thread. Returns false if the OpenVR runtime is missing or no HMD is reachable, so the GUI can warn the user instead of dereferencing a null vr::VRInput() inside vtkOpenVRRenderWindowInteractor::Initialize(). When false, *reason (if non-null) carries a human-readable explanation.

◆ kModelScale

double VRRenderThread::kModelScale = 0.03
staticconstexpr

Per-actor scale applied at add-time. STL files come from the GUI in millimetres, but vtkOpenVRRenderWindow treats 1 VTK unit as 1 metre (PhysicalScale=1.0 by default). Without this conversion an ~3000 mm car ends up 3 km wide in the VR world, way past the default camera far-clip plane (~1 km), and the user sees only the skybox. 0.03 = mm->m with a hero-scale bump so the car reads as larger than life in the play-space. Tune up/down if the model feels too small/large.
